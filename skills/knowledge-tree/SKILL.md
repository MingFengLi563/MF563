# KnowledgeTree - 个人知识树 v1.0.2

## 🚀 快速开始

**添加知识（3 种方式）：**
1. 直接发送链接/图片/文字
2. 说"记录知识"、"添加到知识树"、"整理这个"
3. 发送内容后说"这个也记一下"

---

## 核心功能

1. **知识收集** - 接收链接/图片/文字
2. **智能提取** - 自动总结 + 关键信息摘录
3. **归类整理** - 放入知识树对应分支
4. **关联发现** - 发现与已有知识的联系
5. **应用建议** - 提出实际应用场景
6. **消耗统计** - API 调用次数、token 消耗（每日 0 点自动更新）

---

## 处理流程

### 步骤 1：接收输入

**触发条件：**
- 用户发送 URL 链接
- 用户发送图片（带文字）
- 用户说"记录知识"、"添加到知识树"、"整理这个"

**输入类型处理：**
| 类型 | 处理方式 |
|-----|---------|
| 微信推文链接 | 用 `web_fetch` 抓取内容 |
| 小红书链接 | 用 `web_fetch` 抓取（如失败则提示用户发截图） |
| 图片 | 用 `image` 工具 OCR 识别文字 |
| 纯文字 | 直接处理 |

---

### 步骤 2：提取知识点（v1.0.2 优化）

**提示词模板：**
```markdown
请从以下内容中提取知识点：

【内容】
{content}

【提取要求】
1. 用 200 字以内总结核心内容
2. 提取 3-5 个关键信息点（原文摘录）
3. 标注知识点类型（概念/方法/工具/案例）
4. 给出 1-2 个实际应用场景

【输出格式】
## 📌 核心总结
[200 字以内的精炼总结]

## 🔑 关键信息
• [关键信息 1 - 原文摘录]
• [关键信息 2 - 原文摘录]
• [关键信息 3 - 原文摘录]

## 🏷️ 知识类型
[概念/方法/工具/案例]

## 💡 应用场景
• [场景 1]
• [场景 2]
```

### 步骤 3：归类到知识树

**知识树分支（可扩展）：**
- `product-thinking` - 产品思维
- `ai-tools` - AI 工具
- `marketing` - 营销增长
- `management` - 团队管理
- `tech-trends` - 技术趋势
- `personal-growth` - 个人成长

**归类逻辑：**
1. 分析知识点关键词
2. 匹配最相关的分支
3. 如无匹配，创建新分支
4. 更新 TREE.md 总览

---

### 步骤 4：关联已有知识

**关联发现：**
```bash
# 搜索已有知识点
grep -r "关键词" ~/.openclaw/workspace/knowledge/nodes/
```

**关联类型：**
- `前置知识` - 理解当前知识需要先了解这个
- `延伸知识` - 当前知识的深入或扩展
- `相反观点` - 与当前知识不同的视角
- `相关案例` - 应用当前知识的实际例子

---

### 步骤 5：生成应用场景

**提示词模板：**
```markdown
基于以下知识点，为用户生成应用场景建议：

【知识点】
{knowledge}

【用户背景】
- 职业：产品经理
- 项目：Salesmartly（10,000+ 付费用户）
- 技能：OpenClaw 重度用户

【输出要求】
1. 生成 3-5 个具体应用场景
2. 场景要可执行（有明确第一步）
3. 优先与用户当前工作相关
```

---

### 步骤 6：记录消耗统计（v1.0.2 新增）

**统计数据结构（stats.json）：**
```json
{
  "today": {
    "date": "2026-03-18",
    "apiCalls": 5,
    "tokenUsed": 12500
  },
  "total": {
    "apiCalls": 128,
    "tokenUsed": 345000
  },
  "lastReset": "2026-03-18T00:00:00+08:00"
}
```

**统计记录时机：**
- 每次调用 `web_fetch`、`image`、`web_search` 等 API 后
- 更新 `today.apiCalls` 和 `today.tokenUsed`
- 同步更新 `total` 累计值

**每日 0 点重置：**
- 通过 cron 任务实现
- 重置 `today` 字段为初始值
- 保留 `total` 累计值
- 更新 `lastReset` 时间戳

## 消耗统计实现

### 初始化 stats.json

```bash
mkdir -p ~/.openclaw/workspace/knowledge
cat > ~/.openclaw/workspace/knowledge/stats.json << 'EOF'
{
  "today": {
    "date": "2026-03-18",
    "apiCalls": 0,
    "tokenUsed": 0
  },
  "total": {
    "apiCalls": 0,
    "tokenUsed": 0
  },
  "lastReset": "2026-03-18T00:00:00+08:00"
}
EOF
```

### 更新统计（每次 API 调用后）

```bash
# 增加 API 调用次数
jq '.today.apiCalls += 1 | .total.apiCalls += 1 | .today.date = "2026-03-18"' \
  ~/.openclaw/workspace/knowledge/stats.json > /tmp/stats_tmp.json && \
  mv /tmp/stats_tmp.json ~/.openclaw/workspace/knowledge/stats.json

# 增加 Token 消耗（假设本次调用用了 2500 token）
jq '.today.tokenUsed += 2500 | .total.tokenUsed += 2500' \
  ~/.openclaw/workspace/knowledge/stats.json > /tmp/stats_tmp.json && \
  mv /tmp/stats_tmp.json ~/.openclaw/workspace/knowledge/stats.json
```

### 每日 0 点重置（cron 任务）

**crontab 配置：**
```cron
0 0 * * * bash -c 'DATE=$(date +%Y-%m-%d); jq --arg d "$DATE" ".today = {date: \$d, apiCalls: 0, tokenUsed: 0} | .lastReset = (now | strftime(\"%Y-%m-%dT%H:%M:%S+08:00\"))" ~/.openclaw/workspace/knowledge/stats.json > /tmp/stats_tmp.json && mv /tmp/stats_tmp.json ~/.openclaw/workspace/knowledge/stats.json'
```

**说明：**
- 每天 0 点执行
- 重置 `today` 字段
- 保留 `total` 累计值
- 更新 `lastReset` 时间戳

## 注意事项

1. **隐私保护** - 不上传知识内容到外部 API
2. **去重检测** - 避免重复收录相同知识点
3. **定期整理** - 每周回顾，合并相似知识点
4. **应用导向** - 每个知识点必须有应用场景
5. **统计准确** - 每次 API 调用后及时更新统计
