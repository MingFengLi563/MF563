# api-doc-finder - API 文档搜索助手

## 技能定位

帮助非技术背景的产品经理快速找到官方 API 文档，整理接口清单、参数说明、认证方式，输出结构化摘要。

## 触发条件

当用户提到以下情况时激活此技能：
- "找 API 文档"、"查接口文档"
- "Meta API"、"Google API"、"微信 API"、"钉钉 API" 等平台接口
- "接口参数是什么"、"这个接口怎么用"
- "产品方案需要调用 XX 接口"
- 需要评估接口可行性但找不到文档

## 核心能力

1. **精准搜索** — 使用 searxng 搜索官方文档（优先官方来源）
2. **文档抓取** — 使用 web_fetch 提取文档内容
3. **结构化整理** — 输出接口清单、参数、认证、限流
4. **可行性标注** — 标注接口状态（公开/需审批/已废弃）

## 执行流程

### Step 1: 确认需求
向用户确认：
- 目标平台（Meta、Google、微信、钉钉等）
- 具体功能（发消息、获取用户信息、上传文件等）
- 使用场景（产品方案、技术评估等）

### Step 2: 搜索官方文档
使用 searxng 搜索，关键词示例：
```
site:developers.facebook.com Marketing API
site:developers.google.com Gmail API
site:open.work.weixin.qq.com 企业微信 API
```

### Step 3: 抓取并解析文档
使用 web_fetch 提取文档内容，关注：
- 接口列表（Endpoints）
- 请求方法（GET/POST/PUT/DELETE）
- 请求参数（必填/可选/类型）
- 返回参数（字段说明）
- 认证方式（OAuth/API Key/Access Token）
- 限流规则（Rate Limit）
- 权限要求（Scopes）

### Step 4: 输出结构化摘要

## 输出格式

```markdown
# [平台名称] API 文档摘要

## 📌 文档来源
- **官方文档**: [链接]
- **API 版本**: [版本号]
- **最后更新**: [日期]

## 🔐 认证方式
- **类型**: OAuth 2.0 / API Key / Access Token
- **获取方式**: [简述]
- **有效期**: [如有]

## 📋 核心接口清单

| 接口名称 | 方法 | 路径 | 功能 | 权限要求 |
|----------|------|------|------|----------|
| [名称] | POST | /v1/xxx | [功能描述] | [scope] |

## 📥 请求参数示例

### [接口名称]
```json
{
  "必填字段": "说明",
  "可选字段": "说明"
}
```

## 📤 返回参数示例

```json
{
  "字段名": "类型 - 说明"
}
```

## ⚠️ 限流规则
- **频率**: [X 次/分钟]
- **配额**: [X 次/天]

## 🚦 接口状态
- ✅ 公开可用 / ⚠️ 需审批 / ❌ 已废弃

## 💡 产品建议
- [可行性评估]
- [注意事项]
- [替代方案（如有）]
```

## 注意事项

1. **优先官方来源** — 避免第三方博客、过时文档
2. **标注版本** — API 版本可能影响参数和路径
3. **认证门槛** — 明确告知用户是否需要审批、费用
4. **中文翻译** — 英文文档需翻译关键字段
5. **多接口对比** — 如有多个相似接口，对比差异

## 依赖技能

- searxng（联网搜索）
- web_fetch（网页抓取）

## 示例场景

### 用户问："帮我找 Meta 发送消息的 API 文档"

**执行：**
1. 搜索 `site:developers.facebook.com WhatsApp Business API send message`
2. 抓取官方文档
3. 整理接口：`POST /v17.0/{phone-number-id}/messages`
4. 输出参数、认证、限流、示例

### 用户问："钉钉机器人接口怎么用"

**执行：**
1. 搜索 `site:open.dingtalk.com 机器人 webhook`
2. 抓取文档
3. 整理：webhook 地址、消息类型、签名机制
4. 输出使用示例

## 技能边界

**能做：**
- ✅ 搜索并整理官方 API 文档
- ✅ 输出结构化接口摘要
- ✅ 标注认证和限流要求
- ✅ 提供可行性建议

**不能做：**
- ❌ 直接调用接口测试（需要 api-feasibility-test 技能）
- ❌ 修改或创建 API
- ❌ 绕过认证或限流
