## 现状分析

### 当前页面包含的模块

| 模块 | 内容 | 状态 |
|------|------|------|
| Meta 广告账户授权 | 授权广告账户，支持发送营销消息 | ✅ 有 |
| 账户信息展示 | 广告账户名、ID、关联主页 | ✅ 有 |
| 数据追踪 | 广告数据、消息广告数据 | ✅ 有 |
| 删除确认 | 删除广告账户的二次确认弹窗 | ✅ 有 |
| 关联主页说明 | 授权时获取的主页信息，需重新授权更新 | ✅ 有 |

### 当前授权流程

```
用户点击"授权广告账户"
        ↓
跳转到 Facebook 授权页面
        ↓
用户选择要授权的主页
        ↓
返回 SalesMartly，显示账户信息
        ↓
可删除已授权账户
```

## 核心问题

| 问题 | 风险 | 优先级 | 用户影响 |
|------|------|--------|----------|
| **无授权状态展示** | 用户不知道授权是否过期 | 🔴 高 | 过期后无法发送消息，用户不知情 |
| **无权限说明** | 用户可能拒绝关键权限 | 🔴 高 | 授权失败，功能不可用 |
| **无失败处理** | 授权失败时无引导 | 🟡 中 | 用户不知道如何解决 |
| **无重新授权入口** | 权限变更后无法更新 | 🟡 中 | 需要删除后重新添加 |
| **无使用数据** | 用户不知道授权后的价值 | 🟡 中 | 无法评估 ROI |
| **无操作日志** | 无法追溯操作历史 | 🟢 低 | 团队协作时无法追溯 |

## 优化方案

### 四种状态设计

#### 1️⃣ 空状态 - 未授权

```
┌─────────────────────────────────────────────────────┐
│  📱 Meta 广告账户授权                                │
│                                                      │
│  授权后可实现：                                       │
│  ✅ 向关联主页订阅用户发送营销消息                    │
│  ✅ 追踪广告数据和消息广告效果                        │
│  ✅ 查看广告花费和转化数据                            │
│                                                      │
│  需要的权限：                                         │
│  • ads_management - 管理广告投放                     │
│  • pages_read_engagement - 读取主页互动数据           │
│  • pages_manage_posts - 管理主页帖子内容              │
│                                                      │
│  [📘 查看权限说明]           [🔗 立即授权]            │
└─────────────────────────────────────────────────────┘
```

**设计要点**:
- 清晰说明授权后的价值
- 提前告知需要的权限，降低拒绝率
- 提供权限说明链接，建立信任

---

#### 2️⃣ 已授权状态

```
┌─────────────────────────────────────────────────────┐
│  📱 Meta 广告账户                    ✅ 已授权      │
│  ─────────────────────────────────────────────────  │
│  账户名：SalesMartly Official                       │
│  账户 ID：act_123456789                             │
│  授权时间：2026-03-15 14:30                         │
│  过期时间：2026-06-15 14:30        [🔄 续期]        │
│                                                      │
│  关联主页：                                          │
│  • SalesMartly 主页 (12.5K 粉丝)                     │
│  • SalesMartly CN (3.2K 粉丝)                        │
│                                                      │
│  使用统计（过去 30 天）：                              │
│  📊 发送营销消息：1,234 条                           │
│  💰 广告花费：$5,678                                │
│  👥 触达用户：45,678 人                              │
│  📈 转化率：3.2%                                    │
│                                                      │
│  [📋 查看明细]  [⚙️ 重新授权]  [🗑️ 删除]             │
└─────────────────────────────────────────────────────┘
```

**设计要点**:
- 状态标签清晰可见（绿色✅）
- 过期时间提前 7 天显示提醒
- 使用数据直观展示授权价值

---

#### 3️⃣ 授权失败状态

```
┌─────────────────────────────────────────────────────┐
│  ⚠️ 授权失败                                        │
│                                                      │
│  失败原因：用户拒绝了 ads_management 权限            │
│                                                      │
│  解决方案：                                          │
│  1. 该权限用于管理广告投放，是必需权限                │
│  2. 请重新授权并确保勾选所有权限                      │
│                                                      │
│  [📘 为什么需要这个权限]     [🔄 重新授权]           │
└─────────────────────────────────────────────────────┘
```

**设计要点**:
- 明确告知失败原因
- 提供解决方案，不是只报错
- 解释权限用途，降低用户顾虑

---

#### 4️⃣ 授权过期状态

```
┌─────────────────────────────────────────────────────┐
│  ⏰ 授权已过期                                      │
│                                                      │
│  您的 Meta 广告账户授权已于 2026-06-15 过期          │
│  过期后将无法发送营销消息和查看广告数据               │
│                                                      │
│  [🔄 立即续期]              [🗑️ 删除账户]            │
└─────────────────────────────────────────────────────┘
```

**设计要点**:
- 过期前 7 天开始提醒（站内信 + 邮件）
- 过期后明确告知影响
- 一键续期，无需重新走完整流程

## API 设计

### 1. 获取账户列表

```
GET /api/v1/meta-ad-accounts

Response:
{
  "code": 0,
  "data": {
    "list": [
      {
        "id": 1,
        "account_id": "act_123456789",
        "account_name": "SalesMartly Official",
        "auth_status": "authorized",
        "auth_time": "2026-03-15T14:30:00Z",
        "expire_time": "2026-06-15T14:30:00Z",
        "pages": [
          {"id": "page_1", "name": "SalesMartly", "followers": 12500}
        ],
        "stats": {
          "messages_sent_30d": 1234,
          "ad_spend_30d": 5678.00,
          "reach_30d": 45678
        }
      }
    ],
    "total": 3
  }
}
```

### 2. 发起授权

```
POST /api/v1/meta-ad-accounts/auth-url

Request:
{
  "redirect_uri": "https://app.salesmartly.com/settings/meta-ad-callback"
}

Response:
{
  "code": 0,
  "data": {
    "auth_url": "https://www.facebook.com/v18.0/dialog/oauth?client_id=xxx&redirect_uri=xxx&scope=ads_management,pages_read_engagement,pages_manage_posts"
  }
}
```

### 3. 授权回调

```
GET /api/v1/meta-ad-accounts/auth-callback?code=xxx&state=xxx

处理逻辑:
1. 用 code 换取 access_token
2. 调用 Graph API 获取账户信息
3. 保存到数据库
4. 记录操作日志
5. 重定向到账户列表页
```

### 4. 刷新授权

```
POST /api/v1/meta-ad-accounts/{id}/refresh

Response:
{
  "code": 0,
  "data": {
    "new_expire_time": "2026-09-15T14:30:00Z"
  }
}
```

### 5. 删除账户

```
DELETE /api/v1/meta-ad-accounts/{id}

Response:
{
  "code": 0,
  "message": "删除成功"
}
```

### 6. 获取使用统计

```
GET /api/v1/meta-ad-accounts/{id}/stats?start_date=2026-02-19&end_date=2026-03-19

Response:
{
  "code": 0,
  "data": {
    "summary": {
      "messages_sent": 1234,
      "ad_spend": 5678.00,
      "reach": 45678,
      "conversions": 1456
    },
    "daily": [
      {"date": "2026-03-19", "messages_sent": 45, "ad_spend": 189.50, "reach": 1523},
      ...
    ]
  }
}
```

## 状态机

```
                    ┌─────────────┐
                    │   初始状态   │
                    └──────┬──────┘
                           │ 用户点击授权
                           ↓
                    ┌─────────────┐
              ┌────│  授权中     │────┐
              │    └──────┬──────┘    │
              │           │           │
         用户拒绝    授权成功     网络错误/其他
              │           │           │
              ↓           ↓           ↓
    ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
    │  授权失败   │ │  已授权     │ │  授权失败   │
    └──────┬──────┘ └──────┬──────┘ └──────┬──────┘
           │               │               │
           │ 重新授权      │ 90 天到期      │ 重新授权
           │               ↓               │
           │    ┌─────────────────┐        │
           └───→│   授权过期      │←───────┘
                └────────┬────────┘
                         │
              ┌──────────┼──────────┐
              │          │          │
         用户续期    用户删除   主动撤销
              │          │          │
              ↓          ↓          ↓
         ┌─────────┐ ┌──────┐ ┌─────────┐
         │ 已授权  │ │已删除│ │ 已撤销  │
         └─────────┘ └──────┘ └─────────┘
```

### 状态说明

| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| 授权中 | 正在跳转 Facebook 授权 | 等待回调 |
| 已授权 | 正常可用状态 | 续期、删除、查看数据 |
| 授权失败 | 用户拒绝权限或网络错误 | 重新授权、查看原因 |
| 授权过期 | Token 超过 90 天有效期 | 续期、删除 |
| 已撤销 | 用户在 Facebook 端撤销授权 | 重新授权 |
| 已删除 | 本地删除记录 | 无 |
