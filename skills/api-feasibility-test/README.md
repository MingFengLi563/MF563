# 🧪 api-feasibility-test - 接口可行性测试助手

## 技能简介

帮助非技术背景的产品经理测试 API 接口可用性，获取实际返回参数，截图官方前端页面样式，输出结构化可行性报告。

## 使用场景

- 评估产品方案的技术可行性
- 测试第三方接口是否可用
- 获取接口实际返回数据示例
- 查看官方前端页面长什么样
- 估算开发工作量和风险

## 使用方法

直接描述你的需求，例如：

```
帮我测试 Meta Messenger 发送消息接口能不能用
```

```
这个接口返回什么数据？https://api.example.com/v1/users
```

```
帮我看看这个 API 的前端页面长什么样
```

```
评估一下接入微信支付的可行性
```

## 输出内容

每个可行性测试报告包含：

| 模块 | 内容 |
|------|------|
| 📌 测试概览 | 接口 URL、测试时间、测试状态 |
| 🔐 认证要求 | 认证类型、获取难度、沙箱环境 |
| 📊 测试结果 | 请求示例、返回示例、响应分析 |
| 🖼️ 前端参考 | 官方页面截图（如需要） |
| ⚠️ 风险评估 | 认证/限流/地区/费用风险 |
| 💡 产品建议 | 可行性结论、实现建议、替代方案 |
| 📎 附件 | 文档链接、测试代码、截图 |

## 可行性结论说明

| 结论 | 说明 |
|------|------|
| ✅ 推荐接入 | 接口稳定、文档完善、成本低 |
| ⚠️ 谨慎评估 | 有一定门槛或风险，需权衡 |
| ❌ 不建议接入 | 限制太多、成本过高、有更好替代 |

## 示例输出

```markdown
# Meta Messenger 发送消息接口 可行性测试报告

## 📌 测试概览
- **接口 URL**: https://graph.facebook.com/v21.0/me/messages
- **测试时间**: 2026-03-20 14:30
- **测试状态**: ✅ 可用

## 🔐 认证要求
- **类型**: OAuth 2.0 (Page Access Token)
- **获取难度**: ⭐⭐ 中等
- **沙箱环境**: ✅ 有 (Meta App Dashboard)
- **测试凭证**: 创建测试应用 → 添加 Messenger → 生成 Token

## 📊 接口测试结果

### 请求示例
```bash
curl -X POST "https://graph.facebook.com/v21.0/me/messages" \
  -H "Authorization: Bearer [PAGE_ACCESS_TOKEN]" \
  -H "Content-Type: application/json" \
  -d '{
    "recipient": {"id": "USER_PSID"},
    "message": {"text": "Hello, World!"}
  }'
```

### 返回示例
```json
{
  "recipient_id": "1234567890",
  "message_id": "mid.abc123xyz"
}
```

| 项目 | 结果 |
|------|------|
| 状态码 | 200 OK |
| 响应时间 | ~350ms |
| 数据格式 | JSON |
| 字段数量 | 2 个 |

### 可行性结论
✅ **推荐接入**

- 接口稳定，Meta 官方维护

### 实现建议

## 注意事项

1. **沙箱优先** — 优先使用测试环境
2. **凭证安全** — Token 不记录到日志
3. **限流感知** — 控制测试频率
4. **中文翻译** — 英文内容翻译关键字段
