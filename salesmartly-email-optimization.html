1. 当前功能分析

1.1 现有两种发送路径对比

对比维度 Email 群发 EDM 代发
入口位置营销 → 消息群发 → Email 渠道营销 → 第三方代发
发送方式通过用户集成的邮箱直接发送通过平台 EDM 代发服务
邮箱授权需要授权邮箱账号（IMAP/SMTP）无需授权邮箱账号
费用免费收费（按发送量计费）
发件人用户集成的邮箱地址可自定义发件人邮箱
优势成本低、品牌一致性好送达率高、不消耗邮箱信誉
劣势受邮箱服务商限制需要额外费用
⚠️ 当前问题：
  • 两个功能入口分离，用户需要理解两种发送方式的区别
  • 无法在同一计划中混合使用两种发送方式
  • 不支持按客户归属自动选择发送邮箱
  • 创建群发计划时需要预先决定使用哪种方式

5. 技术实现要点

5.1 数据模型变更

📦 需要新增/修改的字段：
  • customers 表新增 attribution_email_id（关联已集成邮箱）
  • email_campaigns 表新增 send_method（smart/integrated/edm/attribution）
  • email_campaigns 表新增 sender_email_id（指定发件邮箱）
  • email_logs 表新增 send_channel（记录实际发送通道）

5.2 发送通道选择逻辑

💻 核心逻辑伪代码
function selectSendChannel(customer, campaign) {
    switch (campaign.sendMethod) {
        case 'smart':
            // 智能分配：有归属用归属，无归属用代发
            if (customer.attributionEmailId) {
                return { channel: 'integrated', emailId: customer.attributionEmailId };
            } else {
                return { channel: 'edm', emailId: campaign.defaultEdmEmailId };
            }
        case 'integrated':
            return { channel: 'integrated', emailId: campaign.senderEmailId };
        case 'edm':
            return { channel: 'edm', senderAddress: campaign.customSenderAddress };
        case 'attribution':
            if (customer.attributionEmailId) {
                return { channel: 'integrated', emailId: customer.attributionEmailId };
            } else {
                return { channel: 'integrated', emailId: campaign.defaultEmailId };
            }
    }
}
